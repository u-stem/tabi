# URL共有の改善: Web Share Target + OGP自動入力

## 背景

スマホからURLを旅行メンバーと共有するユースケースが使いにくい。
チャット機能は電気通信事業の届出が必要なため避けたい。
Web Share Target でブラウザ共有シートからの導線を作り、OGP自動入力でフォーム入力の手間を減らす。

## スコープ

### スコープ内

- OGP取得APIエンドポイント（共通基盤）
- Web Share Target（manifest + `/share-target` ページ）
- URL入力時のOGP自動入力（候補追加・予定編集フォーム）

### 対象外

- PC向けの共有機能（カスタムプロトコル、ブラウザ拡張など）
- プッシュ通知
- OGPの画像・説明文の取得（タイトルのみ）
- Share Target での旅行新規作成

## 設計

### 1. OGP取得APIエンドポイント

両機能の共通基盤となるサーバーサイドエンドポイント。

**エンドポイント**: `GET /api/ogp?url=<encoded_url>`

**処理**:

1. URLをバリデーション（HTTPSのみ許可）
2. 対象URLのHTMLをfetch（タイムアウト5秒）
3. `og:title` を抽出。なければ `<title>` タグにフォールバック
4. どちらもなければURLのドメイン + パスから生成
5. レスポンス: `{ title: string }`

**セキュリティ**:

- 認証必須（`requireAuth` ミドルウェア）
- SSRF対策: プライベートIPへのリクエストを拒否
- レスポンスサイズ上限: 先頭16KBのみ読む
- 返却値はプレーンテキストのみ（HTMLタグは除去）

### 2. Web Share Target

スマホブラウザの共有シートからsugaraを選んで、候補として即追加する機能。

**manifest.ts への追加**:

```json
{
  "share_target": {
    "action": "/share-target",
    "method": "GET",
    "params": {
      "url": "url",
      "title": "title",
      "text": "text"
    }
  }
}
```

**`/share-target` ページの処理**:

1. クエリパラメータから `url`, `title`, `text` を受け取る
2. `url` がなければ `text` からURL抽出を試みる（アプリによっては `text` にURLが入る）
3. 未ログインなら `/login` にリダイレクト（戻りURLを保持）
4. 未完了の旅行一覧を取得
   - 0件: 「旅行がありません」メッセージ + ホームへのリンク
   - 1件: 即座に候補追加API呼び出し
   - 2件以上: 旅行選択UI表示
5. OGP取得APIで予定名を解決し、候補として POST
6. 完了後、追加先の旅行ページにリダイレクト（トーストで追加通知）

**旅行選択UI**: 旅行名 + 目的地 + 日程を表示するシンプルなリスト。タップで選択し即追加。

**旅行が1件のみの判定**: ステータスが「完了」以外の旅行が1件だけなら選択をスキップして即追加。

### 3. URL入力時のOGP自動入力

既存のフォーム内でURL入力時にOGPからタイトルを自動取得する機能。

**対象フォーム**:

- 候補追加フォーム（`add-candidate-dialog.tsx`）
- 予定編集フォーム（`edit-schedule-dialog.tsx`）

**挙動**:

1. URL入力欄で値が変更され、有効なHTTPS URLが入力される
2. 名前欄が空の場合のみ、OGP取得APIを呼び出す
3. 取得中は名前欄にローディングインジケータを表示
4. 取得成功: 名前欄に自動入力
5. 取得失敗: 何もしない（ユーザーが手動入力）

**トリガー条件**:

- URLが有効なHTTPS URL
- 名前欄が空
- debounce 500ms

**Maps検索との競合回避**:

- Maps検索で名前が入った後にURLを追加 → 名前が既にあるのでOGP取得しない
- URLを先に貼って名前がOGPで入った後にMaps検索 → Maps検索結果で名前が上書きされる（既存の挙動のまま）

**URL入力欄の位置**: 現状のまま変更しない。

## 用語

- 予定（schedule）: 日付に割り当て済みのエントリ
- 候補（candidate）: 日付未割り当てのエントリ（`dayPatternId = NULL`）
